CONFIGURACIÓN OFICIAL DEL PROYECTO SOLINX (OBLIGATORIO)

LEE TODO ANTES DE EMPEZAR.
SI NO SIGUES ESTO EXACTAMENTE, EL PROYECTO PUEDE NO FUNCIONAR.

CONFIGURACIÓN BASE (TODOS IGUAL)

JDK: 17 (Eclipse Temurin)
Servidor: Apache Tomcat 10.1
Java EE Version: Jakarta EE 11 Web
IDE: Apache NetBeans

PARTE 1: INSTALAR JDK 17

Paso 1:
Busca en Google:
Temurin JDK 17 download

Paso 2:
Descarga:
Windows x64

Paso 3:
Instala normal (siguiente, siguiente)

Paso 4:
Verifica:

Abre CMD y escribe:

java -version

Debe decir algo como:

openjdk version "17..."

SI NO → NO SIGAS

PARTE 2: CONFIGURAR JAVA_HOME (OBLIGATORIO)

Esto hace que TODA la computadora reconozca Java.

Paso 1:
Ubica tu JDK, ejemplo:

C:\Program Files\Eclipse Adoptium\jdk-17

Paso 2:
Presiona:
Win + S

Escribe:
Variables de entorno

Selecciona:
"Editar las variables de entorno del sistema"

Paso 3:
Click en:
Variables de entorno

Paso 4:
En "Variables del sistema" → Nueva

Nombre:
JAVA_HOME

Valor:
(Ruta de tu JDK)
Ejemplo:

C:\Program Files\Eclipse Adoptium\jdk-17

Paso 5:
Busca la variable "Path" → Editar

Agrega:

%JAVA_HOME%\bin

Paso 6:
Aceptar todo

Paso 7:
Cierra y abre CMD otra vez

Escribe:

java -version

SI FUNCIONA → LISTO

PARTE 3: CONFIGURAR JDK EN NETBEANS

Abrir Apache NetBeans

Ir a:
Tools → Java Platforms

Click:
Add Platform

Seleccionar carpeta del JDK 17

Verifica que diga:
JDK 17

PARTE 4: INSTALAR TOMCAT 10.1

⚠️ IMPORTANTE:
Jakarta EE 11 usa jakarta.* (NO javax.*)

Paso 1:
Busca en Google:
Apache Tomcat 10.1 download

Paso 2:
Descarga:
.zip (NO instalador)

Paso 3:
Descomprime en:

C:\Tomcat10\

IMPORTANTE:
No cambiar nombre

PARTE 5: AGREGAR TOMCAT EN NETBEANS

Ir a:
Tools → Servers

Click:
Add Server

Seleccionar:
Apache Tomcat or TomEE

Seleccionar carpeta:

C:\Tomcat10\

Terminar

PARTE 6: CONFIGURAR EL PROYECTO SOLINX

Abrir el proyecto

Click derecho → Properties

En "Run":
Server → Apache Tomcat

En "Libraries":
Java Platform → JDK 17

Verificar:
Java EE Version → Jakarta EE 11 Web

⚠️ IMPORTANTE:
Si tu proyecto usa javax.*, NO va a funcionar → debe usar jakarta.*

PARTE 7: EJECUTAR

Click derecho → Run
o presionar F6

Debe abrir:

http://localhost:8080/
ERRORES COMUNES

NO HAY SERVIDOR:
→ No agregaste Tomcat

NO CORRE:
→ No estás usando JDK 17

ERRORS DE IMPORTS:
→ Estás usando javax.* en lugar de jakarta.*

NO RECONOCE JAVA:
→ No configuraste JAVA_HOME

PUERTO OCUPADO:
→ Cambiar puerto o cerrar programa

REGLAS DEL EQUIPO

NO cambiar versiones

TODOS deben usar JDK 17

TODOS deben usar Tomcat 10.1

NO modificar configuración sin avisar

SI FALLA → AVISAR

RESUMEN FINAL

JDK 17 + Tomcat 10.1 + Jakarta EE 11 = FUNCIONA

Cualquier cambio = ERRORES
